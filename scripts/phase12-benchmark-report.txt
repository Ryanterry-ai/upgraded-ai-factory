  Domain Realism (0-100):
    90-100: Recognizable as a real domain app (gym, ecommerce, restaurant)
    70-89:  Has domain elements but some generic parts
    50-69:  Partially domain-aware, mostly generic templates
    0-49:   Generic app with no domain intelligence

  Workflow Realism (0-100):
    90-100: Complete multi-step workflows with domain-specific states
    70-89:  Has workflows but some steps missing or generic
    50-69:  Basic CRUD operations, no workflow orchestration
    0-49:   No workflows, just static pages

  Business Data Consistency (0-100):
    90-100: All entities connected, revenue derived from orders, no placeholders
    70-89:  Most data connected, minor inconsistencies
    50-69:  Some connected data, some placeholder/fake data
    0-49:   All placeholder data, no entity relationships

  Intent Alignment (0-100):
    90-100: Output directly addresses the user's stated problem
    70-89:  Mostly aligned, some generic sections
    50-69:  Partially aligned, significant generic content
    0-49:   Ignores user intent, generates template app

  Production Readiness (0-100):
    90-100: Could ship to real users today
    70-89:  Needs minor fixes before shipping
    50-69:  Needs significant work before shipping
    0-49:   Prototype/demo quality only
